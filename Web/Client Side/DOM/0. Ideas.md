	-  **If you use `href="x"`**: The browser resolves it to the full path of the current page (e.g., `https://lab-id.net/x`).
	    - _Resulting HTML:_ `src="https://lab-id.net/x" onerror=..."` â†’ **Fails** (The breakout doesn't work because the quote `"` is now part of the URL path).
	- **If you use `href="cid:..."`**: Most browsers do not know how to "resolve" a `cid:` scheme to a web path. Because it is an unknown or non-hierarchical scheme, the browser just returns the **exact string** you typed.
	- The `toString()` will be called and everything will be the same, the only difference is how `src` will understand things that starts with `cid:`. in most cases it will reflect it as it is.
```js
var a = document.createElement('a');
a.href = "cid:test";
console.log(a.toString()); // Returns "cid:test"

a.href = "test";
console.log(a.toString()); // Returns "https://your-current-url.com/test"
```
